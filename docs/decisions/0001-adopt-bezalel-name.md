# 0001. Adopt the Bezalel name

## Status

Accepted

## Context

FaithTech Toronto needed a name and identity for the specific function that takes validated hackathon prototypes and turns them into durable, well-architected software for church, ministry, and nonprofit partners — distinct from Create (formation) and the hackathon program (prototyping). This function needed a name that signaled craftsmanship and excellence as central to the mission, not as a secondary concern to good intentions.

## Decision

Name this function **Bezalel**, after the craftsman in Exodus 31:1–5 — the first person Scripture describes as filled with the Spirit of God, specifically for skilled making: "wisdom, understanding, knowledge, and all kinds of skill." The name is chosen to make an explicit claim: that redemptive work can be fully expressed through craftsmanship itself, and that excellence in building is not in tension with the mission but is one of its primary vehicles.

See [`docs/vision/why-bezalel.md`](../vision/why-bezalel.md) for the full rationale and the specific gap this fills relative to a normal dev shop or a one-off hackathon.

## Consequences

- Bezalel is positioned as a craftsmanship-first brand, which raises the bar for what work is allowed to carry the name — see [`docs/model/guardrails.md`](../model/guardrails.md).
- The name requires ongoing explanation for audiences unfamiliar with the Exodus reference; onboarding and README materials carry this context so it isn't lost.
- Naming the initiative after a biblical craftsman commits Bezalel to being explicit about its faith-rooted origin, even while aiming to be credible to skeptical secular technologists.
