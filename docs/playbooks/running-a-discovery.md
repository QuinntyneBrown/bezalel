# Running a discovery

How to run discovery before deciding whether to scope a partner-backed continuation project.

## Purpose

Understand a mission partner's problem, constraints, users, and success criteria well enough to decide whether — and how — a hackathon prototype should continue. Discovery exists to prevent building the wrong thing well; it is not a sales process.

## When to use

- Before scoping any continuation project after a hackathon.
- When a problem surfaced through Create or a hackathon is still framed as a solution request ("we need a website," "can you build us an app") rather than a specific need.

## Steps

1. **Identify the domain steward.** Find the person on the partner side who actually lives with the problem day to day — not necessarily the most senior person. See [`docs/people/roles.md`](../people/roles.md).
2. **Interview, don't prescribe.** Run structured conversations focused on the current state: what do they do today, what breaks, what does it cost them (time, money, trust, missed opportunity)? Avoid jumping to solutions in this step.
3. **Observe if possible.** Where practical, watch the actual workflow (a Sunday service, a volunteer sign-up process, a donor follow-up) rather than relying only on descriptions.
4. **Name the redemptive angle.** Using the frame in [`docs/vision/redemptive-frame.md`](../vision/redemptive-frame.md), articulate specifically who is worse off today and why, in the partner's own terms where possible.
5. **Define success criteria.** Agree with the partner on what "better" looks like in concrete, observable terms — not "a nice app" but "volunteers can sign up in under two minutes without calling the office."
6. **Assess constraints and backing needs.** Timeline, existing systems, technical capacity to maintain what gets built, data sensitivity, practitioner capacity, operating costs, and funding or in-kind support required. The mission partner is not being assessed for an ability to pay a service fee.
7. **Decide: scope, defer, or decline.** Not every discovery leads to a continuation project. If it does, proceed to [`docs/playbooks/scoping-an-engagement.md`](scoping-an-engagement.md).

## Outputs

- A written problem statement and redemptive angle
- Agreed success criteria
- A go / no-go / not-yet recommendation
- If go: enough information to start scoping
