# Glossary

Shared vocabulary used across this repository. If a term used elsewhere in these docs is unclear, it should be defined here.

**Redemptive technology**
Technology that actively repairs, restores, or dignifies something broken, neglected, or excluded — distinct from merely avoiding harm. See [`MANIFESTO.md`](MANIFESTO.md) and [`docs/vision/redemptive-frame.md`](docs/vision/redemptive-frame.md).

**Engagement**
A scoped, time-bounded piece of work in which a Bezalel team takes a specific problem (often a hackathon prototype) from demo quality to durable, well-architected software for a partner. See [`docs/engagements/_template.md`](docs/engagements/_template.md) and [`docs/playbooks/scoping-an-engagement.md`](docs/playbooks/scoping-an-engagement.md).

**Practitioner**
A technologist actively contributing skilled work — engineering, design, product, architecture — to Create, a hackathon, or a Bezalel engagement. See [`docs/people/roles.md`](docs/people/roles.md).

**Domain steward**
The person (often from the partner organization) who holds real knowledge of the problem domain and represents the partner's interests throughout an engagement. See [`docs/people/roles.md`](docs/people/roles.md).

**Create**
FaithTech's ongoing formation program for technologists — the entry point of the flywheel, where practitioners are developed and real problems from churches, ministries, and nonprofits are surfaced. See [`docs/playbooks/create-night-format.md`](docs/playbooks/create-night-format.md).

**Hackathon**
A compressed, time-boxed event where teams prototype solutions against problems surfaced through Create. Hackathons produce candidate prototypes, not finished products. See [`docs/playbooks/hackathon-to-engagement.md`](docs/playbooks/hackathon-to-engagement.md).

**The flywheel**
Bezalel's operating loop: Create → hackathon → Bezalel engagement → feedback into Create. See [`docs/model/the-pipeline.md`](docs/model/the-pipeline.md).

**Partner**
The church, ministry, or mission-driven nonprofit that owns the problem being solved and receives the resulting technology.

**Discovery**
The structured process of understanding a partner's problem, constraints, and success criteria before scoping an engagement. See [`docs/playbooks/running-a-discovery.md`](docs/playbooks/running-a-discovery.md).
