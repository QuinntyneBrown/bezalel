# Bezalel

Bezalel is **not a consultancy**. It is the working name for a [FaithTech Toronto](https://faithtech.com) team exploring what it would look like to bring the product and engineering discipline associated with ThoughtWorks to redemptive technology — while building free of charge for churches, ministries, and mission-driven nonprofits.

The model grows out of FaithTech Create programs and hackathons. Selected projects continue through small, multidisciplinary teams made possible by practitioner time and by funding, tools, mentorship, and other resources from backing partners. The organizations receiving the work are mission partners, not clients.

This repository is not a software product or a catalog of services. It documents the team's working hypothesis, operating model, and playbooks so the model can be tested in practice and improved honestly.

## The flywheel

1. **Create** — FaithTech Create programs develop practitioners and surface real problems from churches and ministries.
2. **Hackathons** — Teams prototype solutions against those real problems in a compressed, high-energy format.
3. **Partner-backed builds** — Selected prototypes continue toward durable, well-architected software at no charge to the mission partner.
4. **Feedback** — Outcomes, learnings, and newly-formed practitioners flow back into Create and the next hackathon cycle.

See [`docs/model/the-pipeline.md`](docs/model/the-pipeline.md) for the detailed hand-offs between stages.
